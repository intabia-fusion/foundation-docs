# Что у нас есть?

Ниже — сжатое и структурированное описание основных модулей и возможностей платформы Foundation (и расширенной версии TraceX), основанное на документации. Это должно помочь быстро понять архитектуру и ключевые сценарии использования системы.

---

## Краткое описание

Foundation — это модульная, открытая, «всё в одном» платформа для управления проектами, знаний и коммуникацией, совмещающая возможности трекера задач, личного/командного планера, совместной редакции документов, системы карточек (Cards), хранилища файлов, виртуального офиса и чата. TraceX — вариант Foundation с дополнительным фокусом на отслеживание соответствия (compliance), включающий контролируемую документацию, подписи и продукты.

---

## Основные принципы и архитектура

- Модульный подход: Workspace запускается с набором модулей (Tracker, Cards, Documents, Drive, Chat, Office, Planner и др.). Некоторые модули можно отключать/включать.
- Пространства (Spaces): контейнеры для объектов (карточек, документов, проектов, дисков и т.д.). У каждого пространства — свои права доступа и роли.
- Роли и разрешения: RBAC (Owner / Maintainer / User), глобальные администраторы и роли на уровне типов пространств.
- Интеграции: GitHub, Google Calendar, Gmail, Telegram и др. — для синхронизации активности, задач, календарей и уведомлений.

---

## 1. Cards — система структурированных карточек (база знаний и не только)

Cards — гибкий фреймворк для моделирования данных (как база данных) и накопления знаний:

- **Types (Типы)** — «чертеж» карточки: определяются свойства (атрибуты), которые будут присутствовать у карточек этого типа (Text, URL, Number, Checkbox, Date, Reference, Select, Multi-select и т.д.). Типы можно наследовать (derived types).
- **Tags (Теги)** — дополнительные наборы свойств, которые можно навешивать на карточки разных типов (flexible mixins).
- **Relations (Связи)** — заранее определённые связи между типами: 1:1, 1:N, N:N; позволяют строить семантические сети между карточками.
- **Parent-child** — отдельный тип иерархической связи (дерево карточек).
- **File Type** — специальный тип для файлов (выделенная область загрузки; используется для управления медиа и артефактами).
- **Spaces для Cards** — наборы карточек с правами доступа, авто-join, владельцами.
- **Views** — таблицы, списки, фильтры и сохранённые представления; сортировка и кастомизация колонок.
- **Activity & Attachments** — лента изменений, комментарии, вложения и история изменений.
- **Конвертация** — из сообщения/комментария можно сделать карточку любого типа.
- Cards — «фундамент» для будущей системы процессов и автоматизации.

Когда использовать Cards: нужен строгий типизированный подход, атрибуты и фильтрация. Для простых заметок и вложенной структуры может хватить Documents.

---

## 2. Documents — командная wiki и редактор

Documents — совместная среда для написания, хранения и обсуждения текстов:

- Коллаборативная редакция (live cursors), markdown-подобный редактор, inline-заметки и комментарии.
- Teamspaces, вложенная структура, backlinks (обратные ссылки).
- Шаблоны документов, экспорт в PDF, история версий.
- Встраиваемые объекты: таблицы, изображения, Mermaid диаграммы, Drawing Board (скетчи) и т.д.
- Внутри документа можно создавать Action Items (чекбоксы) — они превратятся в Task/Action Item для планирования.
- Inline comments — быстрые локальные замечания; Activity — обсуждения, привязанные к объекту.

Когда использовать Documents: свободное текстовое содержимое, long-form документация, заметки и роадмапы.

---

## 3. Chat

Chat — канал коммуникации внутри платформы:

- Поддерживает каналы (channels), DM (direct messages), треды и вложенные под-треды.
- Интеграция с рабочими объектами: документы, issues, pull requests и пр. — активность видна в чате.
- Inbox, фильтрация уведомлений (All, Issues, Documents, Direct messages, Messages).
- Reactions, Bookmark, Pin, Threads, Translate, Save, Reply in thread.
- **Julia AI** — встроенный AI-ассистент: live-transcription (транскрипция встреч), будущие возможности: суммирование, создание action items/documents.
- Почтовая и push-уведомления и интеграция с Telegram (уведомления и отправка сообщений из Telegram).

---

## 4. Tracker — управление задачами и проектами

Tracker — гибкая система трекинга задач в проектах:

- Projects (тип проекта, участники, воркфлоу), Issues (задачи), Sub-issues (subtasks), Features/Epics и пр.
- Task Types и Process States: настраиваемые типы задач с custom workflow. Процессные (status categories) категории позволяют автоматизации.
- Components, Milestones, Labels, Issue Templates, Issue Relations (related, blocked by).
- Создание Issue с асайном на пользователя + статусом "Todo" — автоматически создаёт связанный Action Item в Planner assignee.
- Автоматизация состояний: планирование Action Item переводит issue в "In Progress", закрытие action item переводит issue в "Done".
- GitHub интеграция: двусторонняя синхронизация Issues, Pull Requests, Reviews и Comments. Map репозиториев к проектам; при создании Issue можно синхронизировать с GitHub. Ограничения: текущая интеграция не поддерживает sub-issues, иногда не синхронизируются все метки/дискуссии — см. документацию по интеграциям.

---

## 5. Planner — личное и командное планирование (Action Items & WorkSlots)

Planner — календарь планирования и расписаний:

- Action Items — задачи, назначенные конкретному пользователю; могут быть созданы автоматически (из issue) или вручную.
- WorkSlot — планируемый временной слот для Action Item: drag&drop в календарь.
- Visibility: Visible to everyone / FreeBusy / Only visible to you.
- Team Planner — обзор планов по проектам и участникам; видимость элементов зависит от privacy settings.
- Интеграция с Google Calendar (двусторонняя синхронизация событий и видимости).

---

## 6. Drive — файлохранилище и управление версиями

Drive — файловая система и хранилище:

- Drive Spaces (Records, дополнительные Drives), папки, приватность доступа.
- Загрузка файлов, версии файлов, комментарии, activity.
- Записи митингов (recordings) автоматически сохраняются в Drive.
- Интеграция с карточками/документами: прикрепление, специальные File Types в Cards.

---

## 7. Office — виртуальный офис и видеоконференции

Office — пространство для аудио/видео звонков и виртуальной офисной планировки:

- Типы помещений: индивидуальные офисы (knock/Do Not Disturb), командные комнаты (audio-only), meeting rooms (video).
- Общие возможности: screen sharing, recording, live transcription (Julia AI), guest links (в разработке).
- Meeting Minutes: хранятся расшифровки и записи встреч; политика приватности и доступ к транскрипциям контролируется настройками комнаты.
- Возможность редактирования layout (добавление комнат, этажей и т.п.).

---

## 8. Contacts — управление контактами и сотрудниками

Контакты представлены тремя ключевыми типами: Employee (участники воркспейса), Person (внешние контакты) и Company:

- Контакты хранятся централизованно; поддерживаются custom fields через Classes & Enums.
- Синхронизация GitHub collaborators (Person).
- Возможность создавать Lead/Talent/Contact и конвертировать контакты.
- Взаимосвязь: задачи, вакансии (Recruiting), заявки и т.д. можно связывать с контактами.
- Управление приглашениями: Invite link, Kick/Resend Invite.

---

## 9. Контролируемые документы и TraceX (compliance)

TraceX — расширение для контролируемой документации (Regulated industries):

- Templates (шаблоны документов), Products (версионирование продукта), Review & Approval workflow с e-signatures (21 CFR part 11).
- Document Versioning, Compare versions, Mark as obsolete.
- Training module может быть привязан к ознакомлению сотрудников с документом/изменением.
- Roles & permissions для процессов контроля и документ-ориентированных workflow.

---

## 10. Advanced Settings и конфигурация

- **Types & Tags** — проектирование структуры Cards.
- **Classes & Enums** — добавление кастомных полей для задач и контактов.
- **Space Types & Task Types** — создание кастомных типов пространств и процессов.
- **Roles & Permissions** — Owner / Maintainer / User, Space Admin, Space Type Roles.
- **Text Templates** — часто используемые шаблоны текста для чата/описаний/issue.

---

## 11. Дополнительные модули (включены по желанию)

- **Recruiting** — управление вакансиями, талантами, заявками (disabled by default in many workspaces).
- **Leads** — CRM-подобный модуль (deprecated/disabled by default).
- **Surveys** — возможность создания форм и опросов (experimental).
- **Test Management** — управление тест-кейсами и тестовыми прогонками.
- **Trainings** — создание и назначение обучений и проверок (TraceX).

---

## 12. Интеграции (встроенные и внешние)

- **GitHub** — Issues, PR, Reviews, Comments: двусторонний sync; Map repositories to Projects.
- **Google Calendar** — двусторонняя синхронизация событий, privacy mappings (Public / FreeBusy / Private).
- **Gmail** — отправка/приём писем через страницу контакта (centralized inbox — в планах).
- **Telegram** — получение уведомлений, отправка сообщений из Telegram в чат; взаимодействие с несколькими воркспейсами.

(Примечание: каждая интеграция имеет свои ограничения и текущие особенности — см. документацию по интеграциям.)

---

## 13. Автоматизация и взаимосвязи (примерные сценарии)

- Issue (Tracker) -> assign + status Todo => автоматически создаётся Action Item в Planner assignee.
- Action Item: scheduled => issue переходит в «In Progress».
- Action Item: выполнен => issue переходит в «Done».
- Сообщения / Activity могут быть связаны с документами, issues, карточками и появляться в чате; упоминание @object показывает активность в связанной карточке/документе.
- GitHub PR/Issue activity также отображается и синхронизируется с чатом / tracker при подключении репозиториев.

---

## 14. Статус развития и важные замечания

- Некоторые модули и функции находятся в статусе beta/preview (Surveys, Test Management, Julia AI features и т.д.).
- Некоторые модули по умолчанию отключены (Leads, Recruiting в общих воркспейсах).
- Julia AI: пока транскрипция и базовые функции; будущие планы включают суммирование, создание action items/documents и расширенную автоматизацию.
- Конфиденциальность транскрипций: по умолчанию транскрипции видимы участникам рабочего пространства; можно отключить транскрипцию в настройках комнаты.

---

## Как начать работать: практические рекомендации

1. Настройте воркспейс: создайте пространство, включите нужные модули и пригласите участников.
2. Решите, где что хранить: Documents для лесных заметок и wiki; Cards для структурированных данных и моделей.
3. Создайте базовые Types/Tags и сохранённые Views для карточек: так вы получите быстрый рабочий процесс.
4. Подключите GitHub / Google Calendar / Telegram / Gmail, если требуется интеграция с внешними системами.
5. Используйте Planner и Action Items для управления потоком работы, и помните про автоматизации статусов задач.
6. Если нужно — включите TraceX модули для контроля документации, review/approval и e-signatures.
