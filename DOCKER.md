# Docker для документации Интабия Платформа

Docker контейнер для сборки и хостинга документации Astro с путем `/docs`.

## Быстрый старт

### Сборка образа

#### Используя build.sh (рекомендуется)

```bash
./build.sh [tag]
```

Скрипт автоматически:
- Собирает Docker образ
- Проверяет размер
- Тестирует контейнер
- Проверяет Brotli сжатие

Примеры:
```bash
./build.sh                    # Сборка с тегом latest
./build.sh v1.0.0            # Сборка с тегом v1.0.0
./build.sh test              # Сборка с тегом test
```

#### Ручная сборка

```bash
docker build -t intabiafusion/platform-docs .
```

### Запуск контейнера

```bash
docker run -d -p 8080:80 --name platform-docs intabiafusion/platform-docs
```

Документация будет доступна по адресу: http://localhost:8080/docs/

## Размер образа

- **Сжатый**: ~829 MB
- **Несжатый**: ~800 MB

### Состав образа:
- nginx alpine + brotli (базовый): ~25 MB
- Статические файлы документации: ~805 MB (включая изображения)

## GitHub Actions

Автоматическая сборка и публикация Docker образа настроена в `.github/workflows/docker.yaml`.

### События запуска

- **Пуш в main** → образ с тегом `latest`
- **Создание тега v\*** → образ с тегами версии
- **Pull Request** → сборка без публикации
- **Ручной запуск** → workflow_dispatch

### Версионирование

При создании тега `v1.0.0` образ получает следующие теги:
- `1.0.0` — полная версия
- `1.0` — мажор.минор
- `latest` — последняя стабильная версия

Примеры:
```bash
# Использовать конкретную версию
docker pull intabiafusion/platform-docs:1.0.0

# Использовать последнюю минорную версию
docker pull intabiafusion/platform-docs:1.0

# Использовать последнюю версию
docker pull intabiafusion/platform-docs:latest
```

### Создание новой версии

```bash
# 1. Создать и запушить тег
git tag v1.0.0
git push origin v1.0.0

# GitHub Actions автоматически:
# - Соберет образ
# - Создаст теги: 1.0.0, 1.0, latest
# - Опубликует в Docker Hub
# - Запустит тесты
```

### Необходимые секреты
   - `v1.0.0`, `v1.0` (для тегов версий)
   - SHA коммита

### Необходимые секреты:

Для работы GitHub Actions нужно добавить секреты:
- `DOCKER_USERNAME` - имя пользователя Docker Hub
- `DOCKER_PASSWORD` - пароль или токен Docker Hub

## Производительность

Тесты показывают отличную производительность:

- **Среднее время загрузки страницы**: ~0.9ms
- **Статические ресурсы**: ~1-2ms
- **Brotli сжатие**: до 83% снижение размера (лучше gzip на 15-20%)
- **Gzip сжатие**: fallback для старых браузеров
- **Все страницы возвращают HTTP 200**

### Сравнение сжатия

| Формат | HTML | CSS | JS | Эффективность |
|--------|------|-----|-----|---------------|
| Без сжатия | 25.0 KB | 18.2 KB | 72.9 KB | - |
| Gzip | 8.3 KB | 4.6 KB | 25.0 KB | ~67% |
| **Brotli** | **6.9 KB** | **3.7 KB** | **19.9 KB** | **~78%** |

## Разработка

### Сборка локально

```bash
# Сборка с помощью скрипта (рекомендуется)
./build.sh test

# Или ручная сборка
docker build -t intabiafusion/platform-docs:test .

# Запуск для тестирования
docker run -d -p 8080:80 --name platform-docs intabiafusion/platform-docs:test

# Просмотр логов
docker logs platform-docs

# Остановка
docker stop platform-docs
```

### Обновление документации

1. Внесите изменения в документацию
2. Соберите новый образ
3. Протестируйте локально
4. Создайте PR в main
5. После мержа образ автоматически обновится
