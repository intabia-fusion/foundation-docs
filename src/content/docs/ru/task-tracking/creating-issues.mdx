## Что такое задачи?

Задачи представляют собой конкретные работы, которые необходимо выполнить. Задачи могут быть разбиты на подзадачи для более мелких работ, которые необходимо выполнить для завершения родительской задачи. Задачи организованы в проекты, которые определяют отдельные пространства, содержащие ваши задачи.

> **Примечание:** Всякий раз, когда задача создаётся со статусом «К выполнению» и назначается участнику команды, система автоматически создаёт [активность](/ru/task-tracking/creating-action-items) в Планировщике этого участника, позволяя ему добавить её в своё расписание.

## Создание задачи

Выберите Трекер в левом меню, и убедитесь, что боковая панель открыта, нажав на горизонтальные линии в верхнем левом углу.

Нажмите кнопку `+ Новая задача` в верхней части боковой панели. Откроется диалоговое окно, где вы можете ввести начальные настройки для вашей задачи. После ввода желаемых настроек нажмите `Создать задачу`.

> **Примечание:** Если вы уйдёте со страницы во время создания задачи, вы заметите, что кнопка `+ Новая задача` станет `+ Восстановить черновик`. Нажмите на неё, чтобы снова открыть задачу, над которой работали.

## Настройки задачи

При создании или редактировании задачи вы можете настроить следующие параметры:

**Проект**: В верхнем левом углу диалогового окна вы увидите проект, с которым связана эта задача. Нажмите на название проекта, чтобы увидеть выпадающий список, где вы можете выбрать другой проект для связи с этой задачей.

**Название задачи**: Название вашей задачи.

**Описание**: Описание вашей задачи. Описания могут быть отформатированы с использованием markdown-разметки, включая заголовки, блоки кода, ссылки и многое другое.

**Статус задачи**: Установка статуса задачи «К выполнению» (и добавление исполнителя) автоматически создаст [активность](/ru/task-tracking/creating-action-items) в Планировщике этого участника.

**Приоритет**: Выберите из вариантов: Нет приоритета, Низкий, Средний, Высокий или Наивысший.

**Исполнитель**: Выберите участника, которому назначить эту задачу.

**Метки**: Добавьте метки для этой задачи или создайте новые. Это необязательно, но может быть полезно для сортировки задач позже. Чтобы узнать больше об управлении метками, ознакомьтесь с [этим руководством](/ru/task-tracking/labels).

**Оценка**: Добавьте оценку того, сколько времени потребуется для выполнения этой задачи. Это необязательно, но является полезным инструментом для отслеживания рабочего процесса и планирования.

**Срок выполнения**: Выберите срок выполнения для этой задачи. Это также необязательно.

**Задать родительскую задачу**: Сделайте эту задачу подзадачей существующей задачи. Если у задачи уже есть родитель, просто нажмите значок отмены рядом с родительской задачей, чтобы отсоединить её.

**Прикрепить**: Прикрепите файл, изображение или другой медиафайл к этой задаче.

## Советы по созданию эффективных задач

Существует множество способов настроить задачи в соответствии с потребностями ваших конкретных проектов и рабочих процессов. Однако есть несколько советов, которые следует учитывать при создании задач для вашей команды.

- Задачи, назначенные только одному человеку следует создавать небольшими по объёму. Для более крупных задач используйте подзадачи, чтобы разбить их на части.
- Если задачи не находятся в бэклоге, все новые задачи должны иметь статус «К выполнению», чтобы система создала [активность](/ru/task-tracking/creating-action-items) для назначенного участника команды.
- В соответствии с лучшими практиками, статус задачи не должен требовать ручного изменения с «К выполнению» на «В работе» или «Готово». Статус задачи может автоматически определяться статусом активности назначенного участника команды. Чтобы узнать больше о том, как этот процесс автоматизирован (и почему), прочитайте больше об активностях (ранее называвшихся «todos») <a href="https://huly.io/blog/capturing-dynamic-team-workflows-with-action-items" target="_blank">здесь</a>.
