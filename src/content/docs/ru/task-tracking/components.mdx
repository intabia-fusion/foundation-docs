## Что такое компоненты?

Компоненты — это, по сути, способ категоризации задач внутри проекта, похожий на [метки](/foundation-docs/ru/task-tracking/labels), но с ключевым отличием: задача может принадлежать только одному компоненту, тогда как к одной задаче можно добавить много меток.

Например, компоненты могут использоваться для организации задач по категориям, таким как «UI», «Core», «Backend» и т.д.

## Чем компоненты отличаются от вех?

[Вехи](/foundation-docs/ru/task-tracking/milestones) более полезны при планировании частей проекта, связанных со сроками или фазами работы. Вехи проходят через состояния Запланировано > В работе > Завершено, тогда как компоненты просто используются для категоризации задач вокруг темы или области.

## Использование компонентов

Как и другие объекты в Foundation, компоненты могут иметь описание и вложения. Каждый компонент также имеет раздел активности, связанный с чатом, чтобы участники команды могли вести обсуждения, связанные с компонентом.

### Создание компонентов

Перейдите в `Компоненты` в левом боковом меню Трекера, под проектом, в котором вы хотите создать компонент. Затем нажмите кнопку `+ Компонент` в правом верхнем углу.

Добавьте название и необязательное описание, и выберите участника в качестве Лида, если применимо. Затем нажмите `Создать компонент`.

### Связывание задач с компонентами

На странице компонента вы можете создать новую задачу для привязки к этому компоненту, нажав кнопку `+` в разделе «Задачи». Созданная задача будет автоматически связана с этим компонентом.

Вы также можете связать существующую задачу с компонентом, перейдя на страницу задачи и выбрав компонент в правом боковом меню.

При создании новой задачи из любого места в трекере вы также можете выбрать компонент из выпадающего меню в диалоговом окне создания задачи.

### Использование компонентов для фильтрации задач

Компоненты можно использовать так же, как вехи и метки, для фильтрации и сортировки задач в Трекере. Для более подробных шагов см. [Просмотр задач](/foundation-docs/ru/task-tracking/viewing-issues).
