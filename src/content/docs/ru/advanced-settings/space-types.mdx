## Что такое типы пространств?

Типы пространств — это высший уровень настройки в Foundation. Они определяют структуру вашего рабочего пространства, применяя общий набор ролей и типов задач ко всем пространствам, созданным с этим типом.

Трекер в Foundation основан на типе пространства `Классический проект`. Этот тип пространства поставляется без предустановленных ролей и с одним типом задач (`Задача`) со статусами процесса `Бэклог`, `К выполнению`, `В работе`, `Готово` и `Отменено`. По умолчанию новые проекты, созданные в трекере, будут использовать этот тип пространства.

Однако вы можете захотеть создать собственные типы пространств для моделирования различных типов процессов, которые вы будете отслеживать в своём рабочем пространстве. Например, вы можете создать типы пространств для «Клиентских проектов», «Внутренних проектов» и «Отслеживания ошибок», каждый из которых может иметь свои собственные [роли](/foundation-docs/ru/advanced-settings/roles) и [типы задач](/foundation-docs/ru/advanced-settings/task-types).

## Настройка типов пространств по умолчанию

Все настройки типов пространств находятся в разделе `Настройки` в меню левой боковой панели. Нажмите на `ТИПЫ ПРОСТРАНСТВ`, чтобы увидеть список всех типов пространств, доступных в вашем рабочем пространстве. Нажмите на любой тип пространства, чтобы просмотреть и отредактировать его настройки.

Некоторые свойства и атрибуты пространств в настоящее время не настраиваются. Настройки в первом блоке не могут быть изменены, но вы можете настроить [роли](/foundation-docs/ru/advanced-settings/roles) и типы задач каждого типа пространства [здесь](/foundation-docs/ru/advanced-settings/task-types).

## Создание пользовательских типов пространств

Чтобы создать новый тип пространства, нажмите кнопку `+` рядом с `ТИПЫ ПРОСТРАНСТВ` в меню левой боковой панели настроек.

В левом верхнем углу выберите, для какого вида пространства вы хотите создать тип пространства. Если вы выберете `Трекер`, у вас также будет возможность основать тип пространства на `Классическом проекте`.

После создания пользовательского типа пространства вы можете добавить [роли](/foundation-docs/ru/advanced-settings/roles) и типы задач [здесь](/foundation-docs/ru/advanced-settings/task-types).

## Использование пользовательского типа пространства

Пользовательские типы пространств не могут использоваться для существующих проектов. Чтобы использовать пользовательский тип пространства, вам нужно создать новый проект и выбрать пользовательский тип пространства из выпадающего меню `Тип пространства`.
