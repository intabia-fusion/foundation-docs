Foundation разработан для высокой настраиваемости под любой тип рабочего процесса. Хотя некоторые функции настройки всё ещё разрабатываются, уже доступно несколько опций для настройки вашего рабочего пространства под ваши конкретные нужды.

## Что такое типы задач?

Типы задач определяют виды задач внутри ваших проектов. Тип задачи может иметь свои собственные специальные свойства и состояния процесса.

Чтобы посмотреть на тип задачи по умолчанию, используемый в вашем Трекере, перейдите в Настройки и выберите `Классический проект` в области `ТИПЫ ПРОСТРАНСТВ` в левом боковом меню. `Классический проект` — это [тип пространства](/foundation-docs/ru/advanced-settings/space-types) по умолчанию, используемый в Foundation.

Обратите внимание в разделе `ТИПЫ ЗАДАЧ`, что указан один тип задач: `Задача`. Открытие этого типа задачи покажет его свойства и состояния процесса.

Тип задачи `Задача` имеет состояния процесса `Бэклог`, `К выполнению`, `В работе`, `Готово` и `Отменено`.

## Состояния процесса

Состояния процесса — это этапы, через которые задача проходит в своём жизненном цикле. Каждый тип задачи может иметь свой собственный набор состояний процесса. Система Foundation автоматически перемещает задачи между этими состояниями на основе действий пользователя. Следующие действия вызовут автоматические изменения состояния:

1. **Планирование активности** — Когда исполнитель задачи планирует [активность](/foundation-docs/ru/task-tracking/creating-action-items), связанную с этой задачей, статус задачи изменится с «К выполнению» на «В работе».
2. **Завершение запланированной активности** — Когда исполнитель задачи отмечает [активность](/foundation-docs/ru/task-tracking/creating-action-items) как завершённую (после её планирования в своём Планировщике), статус задачи изменится на «Готово».

Даже если вы настроите свои состояния процесса с использованием других названий (см. ниже), система всё равно распознает эти действия на основе `Категории статуса` состояния процесса. Например, вы можете изменить название состояния процесса с «В работе» на «Активно работаю», и пока `Категория статуса` установлена как «Активный», система всё равно распознает это состояние как то, которое вызывает автоматическое изменение состояния.

## Создание пользовательских типов задач

Чтобы использовать свои собственные настроенные типы задач и состояния процесса, вы можете либо изменить типы задач по умолчанию в Классическом проекте, либо создать новый [тип пространства](/foundation-docs/ru/advanced-settings/space-types). В любом случае это может сделать только владелец или администратор рабочего пространства.

Чтобы создать новый тип пространства, перейдите в настройки и нажмите кнопку `+` рядом с `ТИПЫ ПРОСТРАНСТВ` в левом боковом меню.

Выберите тип пространства, на котором вы будете основывать новый тип пространства; в данном случае вы выберете `Трекер`. Здесь вы можете ввести название для вашего типа пространства (например, «Пользовательский трекер») и выбрать, хотите ли вы основать тип пространства на `Классическом проекте` (рекомендуется). Затем нажмите `Создать`.

Найдите ваш новый тип пространства в левом боковом меню. Здесь вы можете добавить участников к типу пространства и переключить, будут ли новые участники рабочего пространства автоматически добавляться к этому типу пространства.

Нажмите кнопку `+` в разделе `ТИПЫ ЗАДАЧ`. Введите название для вашего нового типа задачи (например, «Задача» или «Тикет»), выберите, может ли этот тип использоваться для подзадач, и нажмите `Создать`.

Шаги для создания и изменения типов задач в Классическом проекте такие же.

## Создание пользовательских состояний процесса

Откройте ваши новые типы задач, чтобы создать пользовательские состояния процесса. Вы увидите заполнители для каждой категории, каждый из которых может быть отредактирован под ваши конкретные нужды.

Нажмите на любое состояние процесса, чтобы отредактировать его. Вы можете изменить название состояния, связанный с ним цвет и категорию статуса. Категория статуса используется для запуска автоматических изменений состояния на основе действий пользователя.

Вы также можете создавать новые состояния процесса. Например, в категории статуса «Активный» вы можете добавить больше состояний, таких как «Начато», «В работе», «Отправлено на проверку» и «На проверке». Вы также можете добавить более конкретные состояния в категорию «Готово/потеряно», такие как «Отменено», «Приостановлено» или «Заблокировано».

## Использование пользовательских типов задач

Чтобы использовать ваши пользовательские типы задач, вам нужно создать новый проект на основе вашего нового типа пространства. В настоящее время невозможно изменить тип пространства, используемый для существующего проекта.

При создании нового проекта выберите ваши пользовательские типы задач из выпадающего меню `Тип проекта`.
